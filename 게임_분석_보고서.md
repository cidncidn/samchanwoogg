## 2. 아키텍처 분석

### 2.1 모듈 의존 관계

```
index.html
  └─ src/main.js (Game 클래스 — 엔트리포인트, 상태 머신, 게임 루프)
       ├─ world.js      ← 성채·지형·조명·소품 (Three.js 씬 구성)
       ├─ player.js     ← 1인칭 이동·생명력·마나 (PointerLockControls)
       ├─ weapons.js    ← 무기 6종·투사체·타격 판정

       ├─ waves.js      ← 공성 사이클(SiegeManager) — 스폰 압박·대공세

       ├─ progression.js← 레벨·XP·무기 등급·데미지 파이프라인
       ├─ skills.js     ← 직업 스킬 + 10티어 스킬 트리
       ├─ npc.js        ← NPC 궁수·마법사·보급병 + AI 동료(AllyManager)
       ├─ shop.js       ← 상점 2종 (보급 상인 / 마법 학자)
       ├─ save.js       ← localStorage 영속 저장 (성/용병 분리)
       ├─ effects.js    ← 피보라·폭발·번개·화면 흔들림 (광원 풀링)
       ├─ audio.js      ← WebAudio 절차 생성 BGM + SFX
       ├─ ui.js         ← HUD·데미지 팝업·정보 패널
       ├─ textures.js   ← 절차 생성 PBR 텍스처 (캔버스 기반)
       └─ models.js     ← GLB 에셋 로더 (해골 캐릭터·소품·HDRI)
```

### 2.2 상태 머신

```
menu → playing ⇄ liveshop (전투 중 상점, E키)
         ↕
       paused (ESC)
         ↓
       gameover (성문 파괴 or 용병 사망)
```

- `playing` 상태에서도 정보 패널(C/I/K)은 전투를 중단하지 않고 오버레이
- `liveshop`은 시뮬레이션(적 AI·스폰·NPC)이 계속 진행되는 실시간 상점

### 2.3 게임 루프 구조

`main.js:loop()` — `requestAnimationLoop` 기반 (Vite HMR 무관)

```
1. dt = clock.getDelta() (상한 50ms)
2. player.update(dt)          — 1인칭 이동·재생
3. weapons.update(dt, t)      — 투사체 물리·충돌·홀드-드로우
4. enemies.update(dt, t)      — 적 AI·진군·공격·HP바
5. npcs.update(dt)            — NPC 궁수·마법사 자동 사격
6. allies.update(dt, t)       — AI 동료 (면 고정 순회)
7. skills.update(dt)          — 운석 낙하·장판 틱
8. siege.update(dt)           — 스폰 제어·일차 전환·대공세
9. world.update(dt, t)        — 횃불 플리커·성문 손상 시각화
10. effects.update(dt, cam)   — 파티클·번개·광원 풀 페이드
11. ui.update(dt)             — HUD 전체 갱신
12. 수리반 HP 회복 + 자동 저장(20초) + 상호작용 힌트
13. 게임오버 체크 (성문 0 or 플레이어 사망)
14. composer.render()          — RenderPass → BloomPass → OutputPass
```

#### 데미지 파이프라인 (Diablo 3 기반)

```
최종 피해 = 무기 기본피해
           × 물리/마법 배수 (상점 강화)
           × 무기 등급 배수 (1.0~2.3)
           × 레벨 데미지 배수 (1 + 주스탯/100)
           × 분산 (0.8~1.2)
           × 치명타 (확률: 5%+등급+레벨, 배율: ×2)
```

- **산탄**: 명중 수치에 반비례(저레벨·저등급 = 넓은 산란)
- **검 to-hit**: 기본 85% + 명중 보정(최대 98%)
- **콤보**: 3.5초 내 연속 처치 시 골드·경험치 배수 상승(최대 +150% at 50콤보)

#### 몬스터 스케일링 (엑셀 시트9~12)

```
몬스터 HP = monStat(일차).hp × 종류가중치 × 등급배수 × 난이도배수 × 밤변주 × 수식어 × 분산
```

- **몬스터 레벨** = 공성 일차 → 지수 스케일링 (HP: `50×1.06^L`)
- **등급**: 일반(88%) → 정예(8%) → 희귀(3.3%) → 미니보스(0.7%)
- **수식어**: 27종 prefix(정예↑만), 4종에 실효과('빠름' speed×1.4, '추가 생명력' hp×1.5 등)
- **난이도**: 10단계(보통~고통VI), URL `?diff=N`으로 선택

## 4. 그래픽·사운드 분석

### 4.1 렌더링 파이프라인

```
Three.js WebGLRenderer (ACES Filmic, exposure 1.3)
  → PCF 소프트 섀도우 (DirectionalLight 2048²)
  → EffectComposer:
      RenderPass → UnrealBloomPass (str 0.55, radius 0.4, threshold 0.8) → OutputPass
```

- **저사양 모드** (`?lowfx`): 그림자·블룸 비활성
- **HDRI 환경광**: Poly Haven 야간 HDR — PBR 재질에 반사·음영 제공
- **절차 생성 텍스처**: 캔버스 기반 석벽(벽돌 패턴)·나무(나뭇결·옹이)·흙(풀·자갈) + 범프맵
- **광원 풀링**: 폭발·번개 시 PointLight 12개 미리 생성, 강도만 점멸 (셰이더 재컴파일 방지)

### 4.2 시각 효과

- **피보라**: 박스 파티클 (중력 낙하 → 바닥 핏자국, 최대 70개)
- **폭발**: HDR 파티클 (블룸 발광) + 광원 풀 섬광 + 화면 흔들림
- **번개**: 지그재그 라인 (7세그먼트) + 착탄 섬광
- **데미지 팝업**: 월드 좌표 → 화면 투영 (금색 일반, 적색 치명타)
- **빙결 시각화**: emissive 0x224466 + 애니메이션 속도 0.45배
- **콤보 표시**: 크로스헤어 우상단 (2~50+, 크기·색상 단계)

## 10. 기술적 특이사항

1. **외부 에셋 파일 0개로 게임 구동 가능**: 텍스처·사운드가 전부 절차 생성. GLB 에셋은 시각 품질 향상용이며 로드 실패 시 절차 생성 모델로 폴백
2. **광원 풀링으로 셰이더 재컴파일 방지**: 폭발·번개 시 PointLight를 동적으로 add/remove하면 Three.js가 전체 셰이더를 재컴파일해 프레임 드롭 발생 → 12개 풀 미리 생성, 강도만 점멸
3. **헤드리스 검증 패턴**: `setAnimationLoop(null)` + `clock.getDelta = () => 0.05` + 수동 `loop()` 펌프로 Puppeteer에서 장기 시뮬레이션 가능
4. **엑셀 RPG 공식 직접 이식**: `RPG_공식설계서_Diablo3기반.xlsx`의 시트 9개 공식을 `formulas.js` 단일 모듈로 이식
5. **게임시작.bat**: ASCII 전용 출력(cmd 한글/chcp 줄깨짐 회피), 포트 재사용 + 준비 폴링
