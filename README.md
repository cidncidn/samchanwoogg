# 최후의 성벽 (Last Bastion)

중세 다크 판타지 1인칭 성벽 방어 게임. 이름 없는 변경 성채에 고용된 마지막 용병이 되어,
북쪽 어둠에서 쉼 없이 밀려오는 언데드 군세로부터 성문을 지킨다. 전투에 끝은 없다 — 성이 버티는 한.

## 1. 환경 정보

| 항목 | 내용 |
|---|---|
| 엔진 | Three.js (r165) |

| 3D 에셋 | KayKit 해골 캐릭터 4종(95개 애니메이션)·던전 소품, Quaternius 활, Poly Haven HDRI — 전부 CC0 |

## 2. 실행 방법

**가장 쉬운 방법 (재부팅 후 포함)**: 프로젝트 폴더의 **`게임시작.bat` 더블클릭** —
서버가 최소화 창으로 뜨고 3초 뒤 브라우저가 자동으로 열린다. 종료는 최소화된 서버 창을 닫으면 된다.

수동 실행:

```bash
cd "D:\Obsidian\Chanwoo\20. Game\castle-defense"
npm install        # 최초 1회 (이후 불필요)
npm run dev        # 서버 기동 → 브라우저에서 http://localhost:5173 접속
```

배포용 정적 빌드:

```bash
npm run build      # dist/ 생성 — 아무 정적 웹서버에나 올리면 동작
```

⚠️ 1인칭 시점은 Pointer Lock API를 사용하므로 `file://` 직접 열기로는 동작하지 않는다. 반드시 로컬 서버(vite)로 실행할 것.

## 3. 조작

| 입력 | 동작 |
|---|---|
| `W A S D` | 성벽 통로 위 이동 |
| `Shift` | 질주 |
| `Space` | 점프 |
| 마우스 | 시점 회전 (1인칭) |
| 좌클릭 | 공격 — **활은 누르고 있을수록 시위가 당겨진다** (풀드로우 시 화살 속도·정확도 +20%), 놓으면 발사. 약 1발/초 |
| `1`~`6` / 휠 | 무기 전환 |
| `Q` | **직업 고유 스킬** (궁수 연발/전사 참격/마법사 운석). 캐릭터 Lv5/10/…/50마다 마법 학자에게서 강화 노드 1개 선택 — 누적될수록 발수·속성·이펙트가 화려해진다 (랭크 0~10, Lv50 궁극기) |
| 우클릭 홀드 | **차지 스킬** — 1.1초 충전("⚡ 해방!") 후 놓으면 강화판 발동 (쿨다운 1.7배): 궁수 = 관통 화살 일제 / 전사 = 대참격 / 마법사 = 운석군 3연속 |
| `E` | 상점 거래 — 보급 상인(북벽 동측) / 마법 학자(북벽 서측). **거래 중에도 전투는 계속 진행된다** |
| `F` | **출성/복귀** — 북벽 동측 밧줄에서 성 밖으로 하강. 지상의 적들이 침입자(나)를 우선 공격한다. 전사는 출성 시 물리 +30% |
| `ESC` | 일시정지 — **AI 동료 소환**(최대 4) / 개발용 "처음부터 다시 시작" 버튼 |

게임 시작 시 직업을 선택한다: **궁수**(화살 200·물리 +20%) / **전사**(생명력 160·물리 +40%) / **마법사**(마나 110·화염구 습득 시작).

### 4.2 적 (6종) — 언데드 해골 군단

| 적 | 모델 | 행동 | 위협 |
|---|---|---|---|
| 해골 졸개 | KayKit Minion (질주) | 성문 직진 | 머릿수 |
| 해골 전사 | KayKit Warrior | 성문 직진 | 중장갑 |
| 해골 주술사 | KayKit Mage | 원거리에서 플레이어 포격 | 플레이어 생명력 |
| 날개 악마 | 절차 생성 | 성벽 위로 날아와 직접 공격 | 검으로 대응 강제 |
| 해골 거병 | KayKit Warrior (대형) | 성문 직진 | 성문 대미지 55/회 |
| 공성 거인 (보스) | KayKit Warrior (3배·적색) | 대공세 시 출현(2일째부터), 성문 강타 + 바위 투척 | 모든 것 |

**희귀도** — 몬스터는 확률표에 따라 수식어를 달고 등장한다: 평범한(75%) · 무서운(12%) · 지독한(5%) · 어두운(3%) · 보기드문(2%) · 잔인한(1%) · 군림하는(1%) · 최악의(0.001%) · 고대의(0.00001%). 희귀할수록 체력·피해·골드·경험치·크기가 배가되고 색이 달라진다(HP바 색·몸체 틴트). 몬스터 레벨 = 공성 일차. 무기 등급도 6단계 희귀도(C 흰 → UC 녹 → R 파 → SR 보라 → SSR 노랑 → UR 빨강)로 정렬된다.

**AI 동료 용병** (ESC → 소환, 최대 4): 플레이어의 80% 화력으로 싸우며, 레벨은 플레이어 경험치의 30% 속도로 성장한다. 소환 순서대로 **북·남·서·동에 1명씩 고정 배치**된다. 각 동료는 배정된 면의 성벽 라인 위에서만(안마당·공중 침범 없이) 플레이어보다 훨씬 느린 속도로 이동하며, 그 면으로 오는 적을 맡는다. 궁수(화살)·마법사(에너지볼트)가 번갈아 합류하는 로컬 협동 — 혼자 4방향을 감당하기 벅찰 때.

전원 스켈레탈 애니메이션(걷기/공격/사망) 재생. 빙결 시 동작 속도도 함께 느려진다.
첫날부터 5종이 혼합 출현하며, 개체마다 크기 편차가 있고 8% 확률로 **정예**(붉은 기운, 체력 2.6배, 골드 3배)가 섞인다.

**4방향 공성**: 성은 4면 정사각 요새(모서리 탑 4기, 성문 4개 — 내구도 공유)이고, 적은 동서남북 무작위 방향에서
몰려와 각자 배정된 성문을 노린다. 플레이어는 외곽 통로를 일주하며 대응한다. 대공세는 한 방향으로 70% 집중된다(배너로 방향 예고).

**지속 공성 구조** (웨이브 없음):
- 게임 1일 = 실시간 10분. 공성 일수가 쌓일수록 위협 수준이 완만히 상승하고 적 구성이 흉포해진다.
- NPC는 **배치된 면만**(사거리 42~45m) 지킨다. 기본 궁수 2명은 북벽 담당 — 나머지 방향은 플레이어가 순회하며 막거나 NPC를 증원해야 한다. 방치하면 성문이 조금씩 깎이지만(실측: 무개입 2분에 약 3,400), 함락까지는 며칠 단위라 자리를 비워도 즉시 무너지지 않는다.
- 성문 게이지 아래 **4방향 위협 카운터**(북/서/동/남 적 수)가 어디가 위험한지 실시간으로 알려준다.
- 2~4분 간격으로 **대공세**가 닥친다 (스폰 폭증 + 2일째부터 보스 출현 확률). 이때가 플레이어의 시간이다.
- 동시 활성 수는 성능 보호를 위해 70마리로 제한 (`waves.js`의 `maxAlive`).
- 보급병은 60초마다 골드·화살을 가져온다.
- 적 체력은 공성 일수당 +8%씩 강해진다.

### 4.3 RPG 성장 시스템 (디아블로·리니지 레퍼런스)

- **캐릭터 레벨 (Lv 1~50)**: 처치 경험치(졸개 4 ~ 보스 150, 정예 3배)로 성장. 레벨당 피해 +2%·명중 +0.5%·치명타 +0.2%, 최대 생명력 +5·마나 +3. **레벨업 시 완전 회복**(디아블로식). HUD 좌하단에 레벨·경험치 바.
- **명중과 분산 — 100% 정확도 폐지**: 화살은 명중 수치만큼 산탄이 퍼지고(저레벨·저등급일수록 빗나감), 검은 to-hit 판정(기본 85%)으로 헛스윙이 난다. 모든 피해는 **0.8~1.2배 분산**(디아블로 min-max).
- **치명타**: 기본 5% + 레벨·등급 보정, 피해 2배 — 붉은 섬광과 큰 숫자로 표시.
- **무기 등급 (리니지식 인챈트)**: 일반 → 마법 → 희귀 → 영웅 → 전설. 등급당 피해 ×1.15~×2.0, 연사 +5~28%, 명중 +8~40%, 치명타 +1~7%. 보급 상인의 "무기 인챈트"로 도박 — 성공률 100/80/60/40%, **실패하면 금화만 날린다** (무기 파괴는 없음). 무기 슬롯 테두리가 등급 색으로 빛난다.
- **데미지 숫자 팝업**: 내가 입힌 피해가 적 머리 위로 떠오른다 (일반 금색 / 치명타 적색 대형).

### 4.4 경제 — 상점 2종

두 상점 모두 실시간(E 키) — 거래하는 동안에도 적은 계속 밀려온다.

- **보급 상인 (동쪽 부스)**: 소모품(화살·기름), 수리반 고용(성문 +15/초 상시 보수, 최대 5),
  **무기 인챈트**(등급 도박), 마나 수정·생명력 강화, NPC 고용(궁수 최대 6 — 2명 기본 배치 / 전투 마법사 최대 4 / 보급병 최대 4).
- **마법 학자 (서쪽 부스)**: 마법 학습 전용 — 마법서 해금(화염구/얼음창/연쇄 번개) + 마법 증폭.

### 4.4 승패

승리 조건은 없다 — 성을 며칠, 몇 달이고 지켜내는 것이 곧 기록이다.

- **용병 사망**: **공성이 1일차로 초기화**된다 (전체 기록 소멸). 새 용병으로 처음부터.
- **성 함락** (성문 내구도 1억 → 0): 그 성의 모든 기록이 끝난다. 새로운 성에서 처음부터.
- 진행 상태는 20초마다 자동 저장(localStorage). 브라우저를 닫아도 성은 그 자리에서 기다린다.

## 5. 코드 구조

```text
castle-defense/
├── index.html        # HUD/상점/메뉴 DOM + 스타일
└── src/
    ├── main.js       # Game 클래스 — 상태 머신, 루프, 그림자/블룸 후처리, 에셋 로드 게이트
    ├── world.js      # 성채·지형·조명·횃불·안개·별하늘 + 소품 배치(decorateWorld)
    ├── models.js     # 외부 GLB 에셋 로더 — 해골 캐릭터 복제·애니메이션, 소품, HDRI
    ├── textures.js   # 절차 생성 텍스처 — 석벽 벽돌·나무결·흙바닥 (캔버스 기반)
    ├── audio.js      # 앰비언트 BGM — 바람·드론·비명 (WebAudio 절차 생성)
    ├── player.js     # 1인칭 조작(PointerLock), 이동 제한, 생명력/마나
    ├── weapons.js    # 무기 정의·뷰모델·투사체·명중 판정 (밸런스 수치는 WEAPONS 배열)
    ├── enemies.js    # 적 정의·AI·절차 생성 모델 (밸런스 수치는 ENEMY_TYPES)
    ├── waves.js      # SiegeManager — 지속 공성: 일차/위협/대공세 사이클, 스폰 압박
    ├── save.js       # 영속 저장 — 성(유지) / 용병(사망 시 소멸) 분리
    ├── skills.js     # 직업 고유 스킬 (Q/우클릭 차지) — 화살 폭풍/참격파/운석 소환, Lv1~5
    ├── progression.js # RPG 성장 — 캐릭터 레벨/XP, 데미지 분산·치명타, 명중·산탄, 무기 등급(GRADES)
    ├── npc.js        # 궁수·전투 마법사·보급병
    ├── shop.js       # 상점 품목·가격 곡선
    ├── effects.js    # 피보라·핏자국·폭발·번개·화면 흔들림
    └── ui.js         # HUD 갱신
```

밸런스 조정 지점: `weapons.js`의 `WEAPONS`, `enemies.js`의 `ENEMY_TYPES`, `waves.js`의 `pickType()`/스폰 간격/`threat`, `shop.js`의 가격 함수. 전부 상수 테이블이라 숫자만 바꾸면 된다.

디버깅: 브라우저 콘솔에서 `game` 전역 객체로 전체 상태 접근 가능 (`game.gold = 9999`, `game.waves.startWave()` 등).

## 6. 로드맵 (장기 비전)

현재 빌드는 싱글플레이 프로토타입. 기획상 최종 목표는 다음과 같다.

1. **멀티플레이 — 접속자 수 비례 전장 규모**: 1인이면 작은 변경 성채, 10~100인이면 대규모 공성전.
   - 기술 경로: Node.js + WebSocket(또는 Colyseus) 서버 권한(server-authoritative) 구조로 전환.
     현재 코드는 입력→시뮬레이션→렌더가 분리돼 있어 시뮬레이션부를 서버로 이전하는 방식.
2. **영구 사망 + 성 단위 매칭**: 사망 시 1레벨 복귀, 같은 성 재입장 불가 → 다른 성에서 재시작.
3. **보급 경제 확장**: 보급병이 실제로 성 밖 필드를 돌아다니며 자원 채집(RTS 일꾼식), 타 성에서 온 물자로 상위 업그레이드, 유저 간 개인 거래.
4. **연출 강화**: 사운드(WebAudio), 시체 잔존, 성벽 파괴 단계 모델, 사다리 등반 적.
