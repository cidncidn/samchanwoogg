# 최후의 성벽 — 개발 로드맵 (연속성 문서)

> ⚠️ 이 문서가 프로젝트의 단일 진실 공급원이다. 새 세션에서 작업을 이어받을 때 반드시 먼저 읽을 것.
> 갱신일: 2026-06-13

## 0. 최종 비전 (사용자 확정, 2026-06-12)

1. **무중단 지속형 온라인 게임** — 스테이지제가 아니라 [성] 단위의 영속 세션. 성이 함락되지 않는 한 세션은 계속되고, 새 유저는 진행 중인 전장에 합류한다.
2. **직업 선택** — 최초 배치 시 궁수/전사/마법사 중 선택. 직업별 역할이 다르며 **전사는 성 밖으로 나가 전투** 가능.
3. **성 밖 자유 이동** — 캐릭터는 성벽 위에 갇히지 않고 성 안팎을 오갈 수 있다.
4. **성 내 다양한 상점** — 실시간 자유 이용. 직업 업그레이드, 무기 구매·강화.
5. **4방향 공성** — 성문 앞 한 방향이 아니라 동서남북에서 적이 몰려온다. 성벽 위에 NPC 궁수·마법사 상주. 신규 유저는 이미 전투 중인 전장에 투입된다. 자리를 비워도 성이 바로 함락되지 않는 스케일(성 내구도 1억 단위, 대규모 몬스터).
6. **다채로운 적** — 단순 근접 일변도가 아닌 체계적·다채로운 공격 패턴. 보스와 전체 난이도는 **동시 접속자 수에 비례**해 조절.
7. **BGM** — 스산한 바람 + 간헐적 비명/처절한 소리. 디아블로풍의 무거운 앰비언트.

- 시뮬레이션과 렌더링이 분리되어 있어 Phase 3에서 시뮬레이션부(enemies/waves/world.gate)를 서버로 옮기는 전략.
- 멀티 전환 전까지 game.state 머신: menu / playing / shop(학습) / liveshop(실시간) / paused / gameover / victory.
- 헤드리스 검증 노하우: `window.game` 핸들, `game.renderer.setAnimationLoop(null)` 후 `game.clock.getDelta = () => 0.05` 패치 + `game.loop()` 수동 펌프, 카메라 자유 배치는 펌프 후 `game.composer.render()`.
- 에셋: `public/assets/` (KayKit 해골·소품, Quaternius 활, Poly Haven HDRI — 전부 CC0). 로더 `src/models.js`.
